When the customer is an AI agent — a new rail for machine-to-machine money

Agentic payments: x402 and the machine-money stack.

Checkout was built for a human holding a card. Autonomous agents instead need to discover, authorize, and settle payments themselves — often sub-cent micropayments for APIs, data, and compute, with no one at the keyboard. A new payment rail is forming to make that possible.

x402 · the open standard. Coinbase revived the dormant HTTP 402 “Payment Required” status code (May 2025): an agent hits a paywalled endpoint → the server returns a 402 with a price, token, and chain → the agent pays in USDC inside an HTTP header → the request is retried and settled on-chain — no accounts, API keys, or human checkout. Now a Linux Foundation standard (Apr 2026) whose members include Visa, Mastercard, Stripe, Google, AWS, Circle, Cloudflare, and Shopify.
What the companies are building
checkout
Stripe
Agentic Commerce Protocol (with OpenAI) powers ChatGPT Instant Checkout; a Shared Payment Token and single-use virtual cards (Issuing) let agents pay within set limits.
settlement L1
Tempo
Stripe + Paradigm payments blockchain — ~100k TPS, ~600ms finality, gas paid in stablecoins; mainnet Mar 2026; its Machine Payments Protocol; backers incl. Visa, Deutsche Bank, OpenAI.
crypto rail
Coinbase
Authored x402; AgentKit gives every agent an on-chain wallet; “Bazaar” lets agents discover and pay x402 services in ~200ms.
the dollar
Circle
USDC is the default settlement asset; its Agent Stack adds policy-bound agent wallets and nanopayments as small as $0.000001.
trust layer
Visa & Mastercard
Intelligent Commerce / Trusted Agent Protocol and Agent Pay / Agentic Tokens add card-network identity and fraud controls to agent transactions.
Four standards converging: ACP (commerce checkout) · Google AP2 (rail-neutral authorization) · x402 (crypto micropayments) · Stripe/Tempo MPP — with the card networks bridging into all of them. Fast-moving; specs & figures as of mid-2026.
