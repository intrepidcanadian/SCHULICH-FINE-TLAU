§6.2 · Ether, gas, and proof-of-stake
03 / 17
If Ethereum is a toll road, ETH is the cost to use it

Buying gas to run smart contracts.

Ether (ETH) is the native token of Ethereum. Users buy “gas” to pay for computation. More complex programs = more gas. Incentivizes efficient code and prevents network spam. Hashing algorithm Ethash; new block every 12–15 sec.

Block reward evolution
5 ETH (genesis) → 3 ETH (late 2017) → 2 ETH (early 2019). No supply cap. ~120M ETH circulating (Apr 2022); 72M from crowdsale.
Proof-of-Stake mechanics
Validators deposit (stake) 32 ETH into a smart contract. One block proposer per 12-sec slot, randomly selected. A committee of validators attests. Checkpoint vote every 32 slots (epoch) for finality.
Slashing
Malicious validators lose their stake (“burned”). Downtime → small penalty (0.1% of tokens). Double signing → severe penalty (5%+).
The Merge · Sept 15, 2022
Two years
Parallel chains
Beacon Chain ran alongside mainnet to test PoS before the switch.
99.95%
Energy cut
PoW mining replaced by validator staking. No more ASIC farms.
12-15s
Block time
Faster than BTC’s 10 min; supports complex contracts.
Why staking changes incentives: validators don’t compete with hash power — they’re chosen randomly in proportion to ETH staked. No energy waste. But it raises the new concern of stake concentration (e.g. Lido controls ~30% of staked ETH).

§6.2 · Technical · world state
03b / 17
Technical deep dive · Ethereum’s data model

Ethereum has accounts — not coins.

Bitcoin tracks UTXOs; Ethereum tracks accounts in a global world state. There are two account types: externally owned accounts (controlled by a private key) and contract accounts (controlled by their own code). The world state is the Merkle root of a state trie that every full node maintains in lockstep.

Account fields · both types
nonce       tx counter (EOA) / contracts created (CA)
balance     ETH held, in wei (1 ETH = 10¹⁸ wei)
storageRoot   root hash of contract’s storage trie
codeHash    keccak256 of contract bytecode (empty for EOA)
An EOA has codeHash = keccak256(""). A contract’s code is immutable after deployment, but its storage mutates on every transaction.
Why not UTXO? Smart contracts need persistent, mutable state — counters, mappings, balances of other tokens. UTXO is stateless by design. Accounts let contracts hold and modify storage between calls.
— Ethereum Yellow Paper · Wood (2014)
World state trie · Merkle Patricia Trie
stateRoot (in block header) | [Patricia trie keyed by address] / | \ 0x123... 0xabc... 0xdef... EOA contract contract { nonce, { .., { .., balance, storageRoot storageRoot 0, ∅ } → trie } → trie }
A Merkle Patricia Trie = radix trie + Merkle hashing. Light clients can prove any account or storage slot with O(log n) hashes. Each block header commits to stateRoot, txRoot, and receiptsRoot.
Hash function
Keccak-256

§6.2 · Technical · EVM & gas economics
03c / 17
Technical deep dive · the world computer

The EVM is a 256-bit stack machine.

Every Ethereum node runs the Ethereum Virtual Machine, a deterministic, sandboxed, 256-bit stack machine. Bytecode is executed one opcode at a time, with each opcode charged a fixed amount of gas. If gas runs out, all state changes revert — but the fee is still paid. EIP-1559 (Aug 2021) replaced the auction-style fee market with a predictable base fee + tip.

EVM opcode gas costs (selection)
ADD / SUB     3 gas   // cheap arithmetic
MUL / DIV     5 gas
SLOAD        2,100 gas (cold) · 100 (warm)
SSTORE      20,000 gas (new) · 5,000 (update)
CREATE      32,000 + bytecode cost
CALL         100 + access list cost
tx base     21,000 gas (every transaction)
Storage is 200× more expensive than computation because every node must replicate it forever. This shapes Solidity idioms: pack into bytes, use events for logs, prefer memory over storage.
EIP-1559 fee formula (post Aug 2021)
tx_fee = gasUsed × (baseFee + priorityFee) baseFee_next = baseFee × (1 + 1/8 × (gasUsed − target) / target) target = 15M gas · cap = 30M gas / block baseFee is BURNED · priorityFee → validator
Base fee rises 12.5% when blocks are full, falls 12.5% when empty — converging to demand. The burn makes ETH deflationary when usage is high. >4M ETH burned since the upgrade.
Gas limit vs. gas price. A user specifies gasLimit (max work) and maxFeePerGas (max price). If execution exceeds gasLimit, EVM throws “out of gas” — state reverts, fee paid in full.
— EIP-1559 · Buterin, Conner, Dudley, Slipper, Norden, Bakhta

§6.3 · Smart contracts & dapps
04 / 17
Crypto LEGO · programmable money

Smart contracts are LEGO blocks.

A smart contract is computer code that executes a transaction without human intervention. Written in Solidity, deployed to a contract address, immutable on-chain. Combine them → dapps. Combine dapps → DAOs. Like LEGO, the only limit is imagination.

A smart contract is a mechanism involving digital assets and two or more parties, where some or all of the parties put assets in and assets are automatically redistributed according to a formula based on certain data that is not known at the time the contract is initiated.
— Vitalik Buterin
Solidity
Object-oriented, statically typed, Turing-complete. Influenced by C++, Python, JavaScript. Runs on the Ethereum Virtual Machine (EVM). Other smart-contract languages: Vyper, Yul, Yul+.
Four dapp rules (GitHub manifesto)
1. Open-source, no third-party intervention. · 2. All data on a public blockchain. · 3. Cryptographic token for access. · 4. Consensus mechanism (PoW or PoS) that generates tokens.

§6.3 · Technical · ERC-20 interface
04b / 17
Technical deep dive · what a token contract actually is

A token is a contract with a mapping.

An ERC-20 token is not a separate currency — it is a single smart contract on Ethereum that keeps a mapping(address => uint256) of balances and exposes six required functions. Wallets and DEXes can interact with any ERC-20 because they all share this interface. This standard is why composability is possible.

Minimal ERC-20 in Solidity
contract Token { // State stored in the EVM forever mapping(address => uint256) public balanceOf; mapping(address => mapping(address => uint256)) allowance; uint256 public totalSupply; event Transfer(address indexed from, address indexed to, uint256 value); event Approval(address indexed owner, address indexed spender, uint256 value); function transfer(address to, uint256 amt) external returns (bool) { require(balanceOf[msg.sender] >= amt, "insufficient"); balanceOf[msg.sender] -= amt; balanceOf[to] += amt; emit Transfer(msg.sender, to, amt); return true; } // approve(), transferFrom(), name(), symbol(), decimals() }
Six required functions
totalSupply()   total tokens minted
balanceOf(a)    tokens held by address a
transfer(to,n)   send n to address to
approve(s,n)   allow spender s to pull n
transferFrom(f,t,n)   spender pulls from f
allowance(o,s)   remaining approval
The approve / transferFrom pattern is what lets a DEX swap tokens without custody — you approve Uniswap to pull X tokens, then call swap. It is also the source of the approval phishing attack vector.
Compile and deploy. Solidity → EVM bytecode (~50 KB for a basic ERC-20) → CREATE opcode deploys it to a new address. Deployment costs ~1M gas (one-time). Every transfer() after that costs ~50K gas.
— ERC-20: Token Standard · Vogelsteller & Buterin (2015)

§6.4 · DAOs & the DAO Hack
05 / 17
A virtual company programmed in code

The DAO Hack & Ethereum’s identity crisis.

A DAO codifies governance — no management team, no paper documents. Slock.It’s “The DAO” raised $150M (12M ETH, 15% of supply) by May 2016. Then hackers exploited the smart contract code and drained $40M. The crypto community had to decide: is the blockchain truly immutable?

Jul 20 2016

Vote: rewrite ledger. ETH/ETC split at block 1,920,000.
The two camps
Purists: blockchain is censorship-free and immutable — even theft must stand. The hacker followed the code.
Pragmatists: 50%+ of holders agreed — rewrite the ledger and return the funds. Buterin sided with this group.
Is a blockchain capable of being modified or not? The central question of the DAO Hack debate — and the moment Ethereum became Ethereum (ETH) and Ethereum Classic (ETC).
— Fintech Explained, Ch. 6
ETH (modified)
Funds returned. Buterin’s vision continues. No supply cap. Powers DeFi today.
ETC (purist)
Cap at 210.7M (10x BTC). Cannot pay gas on Ethereum. Speculative; ~$30 (Apr 2022) vs $1.60 at fork.
Lesson
The vulnerability was not in Ethereum — it was in the DAO smart contract. Code is law only as long as the code is correct. Audit. Audit. Audit.

§6.4 · Technical · validator lifecycle
05b / 17
Technical deep dive · how PoS finalises a block

Propose, attest, finalise, slash.

Time is divided into 12-second slots and 32-slot epochs. One validator proposes per slot; a committee attests. A block is finalised when two consecutive epochs are justified by ~2/3 of staked ETH. Protocol: Gasper (LMD-GHOST fork choice + Casper FFG finality).

Validator state machine
01 · Deposit
Stake 32 ETH
Send 32 ETH to deposit contract 0x0000...00219. Enter activation queue.
02 · Active
Attest every epoch
Sign vote on head + target + source checkpoints. BLS-signed.
03 · Propose
~1 in 8K slots
Selected at random by RANDAO. Build & broadcast block.
04 · Slash / Exit
Burn or withdraw
Voluntary exit → full withdrawal. Equivocation → slash.
Slashing conditions (Casper FFG)
Double proposing — sign two distinct blocks for the same slot. Surround voting — attest in a way that contradicts a prior vote. Penalty: min 1 ETH burned + forced exit. Correlated slashing penalty grows with how many validators are slashed in the same window.
Finality timeline · Gasper
slot: 0 1 2 ... 31 | 32 33 ... 63 | 64 ... epoch: ─── N ──────────| ─── N+1 ──── | N+2 block proposed every slot (12 s) attestations aggregated by committees checkpoint at slot 0 of each epoch epoch N justified → epoch N+1 finalises N ∴ finality ≈ 2 epochs = ~12.8 min
Once finalised, reverting requires 1/3 of staked ETH (~$30B) to be slashed. This is the “economic finality” that PoS sells as stronger than PoW’s probabilistic finality.

§6.5 · Stablecoins
06 / 17
Pegged to $1 · the medium of exchange problem

A stable unit of account.

BTC daily volatility ~4% breaks money’s functions. Stablecoins solve this by pegging to fiat, commodities, crypto baskets, or algorithms. The first appeared in 2014; today the largest, Tether, has a $66B market cap.

Four backings
Fiat-backed (Tether, USDC, BUSD): 1:1 reserves of dollars or USD securities. · Commodity-backed (Digix Gold): physical gold. · Crypto-backed (DAI): overcollateralized with ETH. · Algorithmic (TerraUSD): no reserves — supply algorithm.
Centralized vs. decentralized
Tether (USDT) is run by the Bitfinex founders — centralized custody, regulator-fined for misrepresenting reserves. DAI is minted by the MakerDAO smart contract — decentralized, governance-token controlled.
In 2019 Tether admitted only $0.74 of each USDT was backed by cash and securities. USDT briefly dropped to $0.92. Regulators fined the founders for misrepresentation — yet Tether remains the largest stablecoin.
— Fintech Explained, Ch. 6
Case study · TerraUSD & LUNA collapse (May 2022)
UST market cap peaked at $60B. The Anchor protocol promised 20% APR to UST depositors — unsustainable yields that attracted speculative capital. Sister token LUNA crashed $119 → $0 in days. UST de-pegged to $0.10. $45B in market cap destroyed; rippled to $300B across crypto. Founder Do Kwon faces criminal charges.

§6.6 · Decentralized finance
07 / 17
Web3 · the internet of value

DeFi: $40B locked.

Decentralized finance is a marketplace and infrastructure that replicates banking, brokerage, asset management, and insurance using smart contracts on a blockchain. No intermediaries. Always open. Permissionless. The metric: Total Value Locked.

Peak TVL: $175B (2021)·Year-end 2022: $40B·5% of all crypto
Fig. 6.3 · TVL in DeFi 2018–2022Source: DefiLlama
Why Ethereum dominates
~60% of DeFi TVL runs on Ethereum (year-end 2022). Tron 10.8%, BSC 10.5% trail. Ethereum’s ERC-20 standard, Solidity, EVM, and the deepest dev community created insurmountable network effects.
The paradigm debate
Transformational view: DeFi will dramatically alter how financial markets operate — disintermediating banks, brokers, and clearinghouses. Traditional view: DeFi will co-exist with the old system but never dominate it — regulation, trust, and scale favor incumbents.
Decentralized finance has always been a big part of what I hoped to see people build on Ethereum.

§6.7c · Case study · MakerDAO
10 / 17
A decentralized central bank

Mint DAI against ETH collateral.

MakerDAO is the largest DeFi protocol — $6B TVL at year-end 2022. Deposit ETH or ERC-20 tokens, mint DAI (a $1-pegged stablecoin) against them. Stay above the 200% collateralization ratio or face liquidation.

CDP mechanics
Deposit $100 of ETH → mint up to $50 of DAI (50% LTV / 200% collateralization). Pay an interest rate on the loan that algorithmically adjusts to maintain the $1 DAI peg.
Why borrow DAI?
Get liquidity without selling your ETH. Avoid capital-gains tax events. Lever up by re-depositing DAI elsewhere. Use DAI for stable USD transactions on-chain.
Decentralized central bank
No government. No board. The Maker governance token holders vote on rates, collateral types, and parameters. The smart contract enforces the rules. Liquidations are automatic.
Abracadabra (Magic Internet Money): the #2 CDP protocol. Accepts interest-bearing tokens as collateral — letting users stretch their collateral twice and create on-chain leverage. Higher returns, higher liquidation risk.

§6.7d · Yield, derivatives, bridges
11 / 17
Composability · DeFi LEGO in action

Lido, Synthetix, WBTC.

Liquidity staking, derivatives, and bridges expand DeFi’s scope beyond simple swaps and loans — toward a full-stack alternative financial system.

Lido · liquidity staking
Double-dipping rewards
Stake ETH → receive stETH (1:1). Then stake stETH elsewhere across DeFi for additional yield. $18.7B TVL within 18 months of launch. The leading liquidity-staking protocol.
Synthetix · derivatives
On-chain synths
Buy sBTC for BTC exposure. sUSD, sETH, synths of fiat and equities. Trade derivatives for a 0.30% fee — no broker, no clearing house. TVL $1.1B (spring 2022).
WBTC · bridge token
Bitcoin on Ethereum
ERC-20 token backed 1:1 with BTC. Brings Bitcoin liquidity into Ethereum DEXes, wallets, and lending pools. Single node handles both networks.
Yield farming · chasing APY
Maximize returns by moving liquidity across protocols. Deposit into Curve, flip the LP token to Convex Finance, restake the new LP token elsewhere. Higher yield, higher complexity, higher risk — smart-contract exploits compound across layers.
Most blockchains are closed by design — they lack interoperability. Bridges break that wall, but every bridge is also a new attack surface.

§6.7e · Tokenomics
11b / 18
Token economics · theoretical foundations

Tokens vs. equity: when does a coin beat a share?

Tokenomics studies how token-based capital raising differs from traditional equity financing — and when tokens may be the better vehicle. The research builds theoretical foundations for a token-based economy.

01 · Token pricing
Demand-driven, not DCF
Equilibrium token price is determined by total volume of user demand — not discounted cash flows. Feedback between adoption and token price accelerates growth and dampens volatility.
02 · Tokens vs. equity
Agency & moral hazard
Tokens dominate equity for start-ups where entrepreneurial effort is crucial and signaling quality to outside investors is hard. Optimal design: ICO crowdsale + revenue-sharing agreement.
03 · Commitment device
Blockchain solves moral hazard
Tokenization prevents platform owners from diluting users’ stakes. Founder payoff depends on profit; token investor payoff depends on transaction volume. Incentives align.
Case example · Filecoin (FIL)
Digital marketplace for data storage. Users pay in FIL tokens; miners post FIL as collateral. As the user base grows, counterparties are easier to find, increasing the usefulness and value of FIL. Network effects drive token price — not fundamentals.
The supply paradox
Miner compensation increases token supply, lowering investor willingness to pay — limiting utility tokens as a financing alternative. Counter-solution: token burning reduces oversupply and supports value.
Decentralized governance provided by issuing tokens on a blockchain reduces the moral hazard problem between insiders and outsiders and aligns their incentives, encouraging adoption of the platform by users.
— Fintech Explained, Ch. 6

§6.8 · R3 Corda · permissioned DLT
12 / 17
The pragmatist’s blockchain

A DLT built for regulated finance.

David Rutter (ex-ICAP CEO) saw blockchain’s potential for back-office settlement — without the hype. R3’s permissioned DLT validates transactions individually with trusted notaries. No mining. No tokens. Just lower costs for the banks who funded it.

Funding model — banks as members
42 banks joined by year-end 2015. Each paid up to $5M annual membership and seconded staff. $100M+ raised without giving up equity. Eliminated the “crypto VC” problem — banks underwrote their own back-office disruption.
Five Corda choices vs. Ethereum
1. Permissioned, not public. · 2. Validates transactions individually, not in blocks. · 3. Notaries, not mining. · 4. Data shared only with counterparties. · 5. Smart contracts run inside the JVM.
One estimate put the back-office savings at $15–$20B/year. By year-end 2022 hundreds of regulated institutions — central banks, commercial banks, fintechs, exchanges — were using Corda.
